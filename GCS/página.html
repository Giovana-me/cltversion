Gerenciamento de Configuração de Software:

Controle de Versão

O controle de versão é uma prática que permite rastrear e gerenciar alterações em um projeto de software. Ele identifica em qual versão está o software e quais são as características de cada versão. Um sistema de controle de versão de código-fonte é fundamental para a integração contínua.

Controle de Mudanças

O controle de mudanças envolve testar as alterações assim que são realizadas. Ele garante que as mudanças sejam avaliadas antes de serem integradas ao projeto. Essa prática é essencial para manter a qualidade do código e evitar problemas futuros.

Integração Contínua

A CI é uma prática ágil e de DevOps que automatiza a integração de alterações de código de vários contribuidores em um único projeto de software. Os desenvolvedores mesclam frequentemente as alterações em um repositório central, onde builds e testes são executados automaticamente. Um sistema de controle de versão é o ponto crucial do processo de CI. Além disso, verificações como testes automatizados de qualidade de código e análise de estilo de sintaxe complementam a integração contínua
